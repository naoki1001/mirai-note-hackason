# Mirai Note - プロジェクト仕様書

## 1. コンセプト
「未来のことしか呟けないSNS」プロトタイプ（ハッカソン用、実装目標6時間）。
ユーザーは未来の日付や期間を指定して投稿を行う。
現在の出来事ではなく、未来への期待、目標、予想をタイムカプセルのように残すサービス。ハッカソンのテーマである「50年後」を最大スケールとして設定する。

## 2. コア機能（2つの投稿タイプ）
- **未来予想 (Prediction):**
  - 投稿直後からタイムラインに即時公開される。「来週こうなる」「10年後こうなる」といった予想や宣言を共有する。
  - タイムラインでは最初から本文が読める。開放的でポジティブなデザイン。
- **タイムカプセル (Capsule):**
  - 指定した未来の日付が来るまで本文は完全に「封印（マスキング）」される。
  - 自分へのメッセージや非公開の目標。すりガラスで美しく守られているような演出を行う。

## 3. 画面構成・要件 (MVP)
### 画面1: 未来投稿作成フォーム
- 投稿タイプの選択（未来予想 / タイムカプセル）
- **ターゲット日選択:** 1週間後, 1ヶ月後, 1年後, 3年後, 10年後, 50年後, 任意日付
- **カテゴリ選択:** 自分の未来, 仕事の未来, 技術の未来, 生活の未来, 社会の未来, 夢・目標
- 本文入力
- 投稿ボタン

### 画面2: タイムライン（メイン画面）
- 直近の未来から50年後まで、未来予想とタイムカプセルが混在して流れる。
- **未来予想カード:** 本文表示。予想と現在日時、いつの予想なのかが表示される。これはシンプルにつぶやかれた順で表示。
- **タイムカプセルカード:** 指定日まで本文は隠蔽。すりガラス効果（`backdrop-blur`）で「美しく封印されている状態」を演出。いつの予想なのかは見える。
- **振り返りアクション:** ターゲット日を過ぎた投稿には、「振り返り（答え合わせ）」用のボタン（叶った / 外れた / まだ途中）が出現する。最近開封された順みたいに表示

### 画面3: デモ用タイムマシン（ハッカソン審査用）
- 画面隅に配置するデモ用のタイムスキップ機能。
- ボタン操作で仮想現在時刻（TimeContext内）を進め、タイムカプセルが開封される演出や、未来予想の答え合わせ期日が到来する様子を実演できる。

## 4. 主要データ項目
- `id`: string (一意の識別子)
- `type`: 'prediction' | 'capsule' (投稿タイプ)
- `createdAt`: number (作成日時のタイムスタンプ)
- `targetDate`: number (未来の指定日のタイムスタンプ)
- `category`: string (カテゴリ名)
- `content`: string (本文)
- `reviewStatus`: string (振り返りステータス)

## 5. 開発時の初期設定要件
- 初回起動時（`localStorage` が空の場合）、デモ用に過去から未来までのサンプルデータ（未来予想とタイムカプセル）を自動生成すること。
